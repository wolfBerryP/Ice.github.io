单例模式的两种写法

单例模式

单例模式，是一种常用的软件设计模式。在它的核心结构中只包含一个被称为单例的特殊类。通过单例模式可以保证系统中，应用该模式的一个类只有一个实例。即一个类只有一个对象实例。

1.饿汉式

  • 思路：

    1.私有化构造函数，让别人无法随意创建对象

    2.在本类中创建自己唯一的实例，然后用一个静态的变量进行保存

    3.创建一个静态的公共方法，返回这个唯一实例的引用，供别人获取这个对象

class Singleton{
    //1.私有化构造函数
    private Singleton(){

    }
    //2.在本类中创建自己的唯一实例，然后用一个静态的私有的变量保存
    private static Singleton instance = new Singleton();

    //3.创建一个静态的公共方法，返回这个唯一实例的引用，供别人获取这个对象
    public static Singleton getInstance(){
        return instance;
    }
}

这种写法的问题在于在类加载的时候，就已经创建了这个类的唯一实例，不管有没有使用它，这个内存都已经被占用了，有可能造成空间浪费。

2.懒汉式

注：针对上述问题的解决办法

  • 如果在调用者第一次调用的时候在创建这个唯一实例，就可以解决问题了
class Singleton2(){
    //1.私有化构造方法
    private Singleton2(){

    }
    //2.为了避免空间浪费，在类加载的时候不创建这个类的实例，而是等到第一次调用的时候再创建
    private static Singleton2 instance = null;
    //3.在调用的时候检查instance里面有没有这个类的实例，如果没有就创建一个，保存到变量中
    public statci Singleton2 getInstance(){
        if(instance=null){
            instance = new Singleton2();
        }
        return instance;
    }
}

注：懒汉式在多线程中是不安全的

多层继承里面，子类可以通过直接继承的父类得到间接继承的父类的成员
